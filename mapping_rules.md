# 上线内容与平台变更步骤映射规则

## 通用规则
- 每个来源Sheet通过"步骤定义"划分为一个或多个平台步骤
- 筛选规则格式：操作类型为"X" | 操作类型为"X"或"Y" | 某列非空 | 无
- 不匹配任何步骤定义的行归入"其他"步骤

## 应用配置
- 来源Sheet: 应用配置
- 说明: 应用级别的环境变量和配置项管理
- 步骤定义:
  - 平台步骤名: 数据源新增
    - 筛选规则: 操作类型为"数据源新增"
    - 操作描述: 在应用配置管理平台中新增数据源配置
  - 平台步骤名: 数据源删除
    - 筛选规则: 操作类型为"数据源删除"
    - 操作描述: 在应用配置管理平台中删除数据源配置
  - 平台步骤名: 配置新增
    - 筛选规则: 操作类型为"配置新增"
    - 操作描述: 在应用配置管理平台中新增应用配置项
  - 平台步骤名: 配置修改
    - 筛选规则: 操作类型为"配置修改"
    - 操作描述: 在应用配置管理平台中修改应用配置项
  - 平台步骤名: 配置删除
    - 筛选规则: 操作类型为"配置删除"
    - 操作描述: 在应用配置管理平台中删除应用配置项
  - 平台步骤名: 配置发布
    - 筛选规则: 操作类型为"配置发布"或"配置取消发布"
    - 操作描述: 在应用配置管理平台中发布或取消发布配置

## 容器配置
- 来源Sheet: 容器配置
- 说明: Docker容器级别参数配置
- 步骤定义:
  - 平台步骤名: 容器配置
    - 筛选规则: 无
    - 操作类型: 新增、修改、删除、发布
    - 操作描述: 在容器管理平台中，对指定部署单元的容器参数进行增删改及发布操作

## MQS配置
- 来源Sheet: MQS配置
- 说明: 消息队列Topic和授权管理
- 步骤定义:
  - 平台步骤名: MQS配置
    - 筛选规则: 无
    - 操作类型: 新增、删除、新增授权、删除授权
    - 操作描述: 在MQS管理平台中，创建或删除Topic，以及对应用进行Topic的消息收发授权管理

## ROMA任务与事件
- 来源Sheet: ROMA任务与事件
- 说明: ROMA集成平台任务和事件管理
- 步骤定义:
  - 平台步骤名: ROMA任务
    - 筛选规则: ROMA任务名列非空
    - 操作类型映射: 任务上线→上线, 任务下线→下线, 任务修改保存→修改, 任务历史版本删除→删除, 任务历史版本回滚→回滚
    - 操作描述: 在ROMA平台中，对集成任务进行上线、下线、修改保存、历史版本删除及回滚操作
  - 平台步骤名: ROMA事件
    - 筛选规则: ROMA事件名列非空
    - 操作类型映射: 事件部署→部署, 事件取消部署→取消部署, 事件授权→授权, 事件取消授权→取消授权
    - 操作描述: 在ROMA平台中，对事件进行部署、取消部署、授权及取消授权操作

## 上线代码包清单
- 来源Sheet: 上线代码包清单
- 说明: 按包类型分别作为实施总表中的独立任务行
- 步骤定义:
  - 平台步骤名: 上线代码包清单
    - 筛选规则: 无
    - 操作描述: 将数据库脚本、后端代码包、前端代码包上传至部署平台，通过自动部署方式部署到对应的部署资源

## 上线制品包
- 来源Sheet: 上线制品包
- 说明: 按变更应用和部署目标分组，执行顺序相同的制品包在同一个步骤内并行部署。每个唯一的(变更应用, 部署目标, 执行顺序)组合对应一个独立的平台变更步骤
- 部署目标类型: OPENGAUSS, DWS, ADS, WCM
- 各部署目标的有效列（仅这些列有值，其他列为空）:
  - OPENGAUSS/DWS: 发布库路径, 账号, schemaName, 执行顺序, 变更应用
  - ADS: 发布库路径, 部署单元, 实例数, 基础镜像, 执行顺序, 变更应用
  - WCM: 部署单元, 执行顺序, 发布库路径, 发布类型, 变更应用
- 分组规则:
  - 第一级: 按变更应用分组
  - 第二级: 按部署目标分组
  - 第三级: 按执行顺序分组，执行顺序相同的行在同一个步骤内并行执行
- 步骤定义:
  - 平台步骤名格式: "{变更应用} - {部署目标}部署"
  - 若同一(变更应用, 部署目标)有多个不同执行顺序，步骤名后追加"（顺序{执行顺序}）"以区分
  - 例如: "应用A - OPENGAUSS部署（顺序1）"、"应用A - OPENGAUSS部署（顺序2）"、"应用B - ADS部署"
  - 筛选规则: 按上述三级分组规则，将每行分配到对应的(变更应用, 部署目标, 执行顺序)步骤
  - 操作描述: 按部署目标类型执行对应的制品部署操作。OPENGAUSS/DWS为数据库脚本部署，ADS为应用容器部署，WCM为内容管理部署
